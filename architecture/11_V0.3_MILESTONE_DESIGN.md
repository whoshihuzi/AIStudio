## M9 — Workspace Foundation

### Goal
Establish the WorkspaceProvider and IPC layer. No UI.

### Scope
- `src/main/workspace/types.ts` — FileStat, DirectoryEntry, FileContent, SearchMatch types
- `src/main/workspace/WorkspaceProvider.ts` — implements `IWorkspaceProvider`
  - `readFile()`, `exists()`, `stat()`, `listDirectory()`, `glob()`
  - Path security: all paths relative to workspace root
- `src/main/index.ts` — 6 IPC handlers: `workspace:read`, `workspace:list`, `workspace:stat`, `workspace:glob`, `workspace:exists`
- `src/preload/index.ts` — `window.api.workspace.*` bridge
- `src/renderer/env.d.ts` — workspace types

### NOT in M9
- No search (`searchText` skeleton only)
- No write operations (`writeFile`, `createFile`, `delete`, `rename` deferred to M12)
- No UI

### Verification
- typecheck + build pass
- WorkspaceProvider can read a known file (e.g., `package.json`) via smoke test

## M10 — Workspace Explorer

### Goal
Add a file tree widget to the Dashboard.

### Scope
- `src/renderer/components/WorkspaceExplorer.tsx` — collapsible file tree
  - Renders `DirectoryEntry[]` from `workspace:list`
  - Click file → `workspace:read` → display content preview
  - Auto-expand current directory
- `src/renderer/stores/workspace.ts` — WorkspaceStore (Zustand)
  - `rootEntries`, `activeFile`, `openFiles[]`
- Dashboard layout: WorkspaceExplorer as 6th widget (between ProjectBrain and WhatNext)

### NOT in M10
- No file editing
- No drag-and-drop
- No context menu
- No file icons

### Verification
- Dashboard shows file tree
- Clicking a file shows its content
- File tree reflects real filesystem state

## M11 — Search & Index

### Goal
Add full-text search to the WorkspaceProvider and Dashboard.

### Scope
- `WorkspaceProvider.searchText()` — full implementation
  - `maxResults`, `includePattern`, `excludePattern`, `caseSensitive`
  - Uses ripgrep or grep via child_process
- `src/renderer/components/SearchBar.tsx` — Dashboard search bar
  - Type query → results appear below
  - Click result → open file at matching line
- ContextBuilder: include search results in Agent context when relevant

### NOT in M11
- No symbol index (TypeScript language service deferred to v0.4)
- No incremental index
- No fuzzy search

### Verification
- Search "IAgentRuntime" → finds `src/renderer/runtime/types.ts`
- Search limited to 100 results
- Search respects include/exclude patterns

## M12 — Code Manipulation Foundation

### Goal
Enable write operations through WorkspaceProvider. Prepare for Diff.

### Scope
- `WorkspaceProvider.writeFile()`, `createFile()`, `createDirectory()`, `delete()`, `rename()`
- Write audit: every write logged to `workspace/.history/`
- `WorkspaceProvider.getDiff()` — skeleton (delegates to future DiffProvider)
- Dashboard: "Modified" badge on files in WorkspaceExplorer

### NOT in M12
- No Diff view widget (requires Monaco editor integration)
- No Accept/Reject buttons (requires agent-generated diff format)
- No patch application (requires DiffProvider implementation)

### Verification
- Agent can create/rename/delete files through WorkspaceProvider
- Write audit trail exists in `workspace/.history/`
- Dashboard reflects file changes after write

## Future Compatibility (M13+ / v0.4)

All interfaces frozen in M9-M12 are forward-compatible with:

| Feature | Reuses | New |
|---|---|---|
| Diff View | `workspace:read` + `FileContent` | `DiffProvider.getDiff()` |
| Accept/Reject | `workspace:write` | `DiffProvider.applyPatch()` |
| Symbol Index | `workspace:list` + `SearchResult` | `SymbolProvider` (TS language service) |
| Knowledge Graph | `SearchResult` | `GraphProvider` (cross-file references) |
| Multi-Agent | `workspace:read/write` | File locking, conflict detection |
| Remote Workspace | `IWorkspaceProvider` interface | `RemoteWorkspaceProvider` (SSH/WebSocket) |

**Key design property**: `IWorkspaceProvider` is an interface. Future backends (RemoteWorkspaceProvider, VirtualWorkspaceProvider) implement the same interface. No IPC change. No Renderer change.
